---

提交所有变更（纯本地操作）。未指定子代理时由主代理直接执行；指定子代理时委托子代理执行。

## 提交要求

无论由谁执行，提交都须满足：

- 按可审计粒度提交，提交信息清晰，且只提交应入库文件
- 提交信息满足 Conventional Commits 规范
- 运行修改内容相关的最小范围测试，如果失败需要修复并提交

## 主代理流程（默认）

1. 先检查 `git status --porcelain` 再决策。
2. 若发现应忽略文件/临时文件：征求用户意见后决定 1 纳入提交 2 保留并更新 `.gitignore` 3 直接删除。
3. 按「提交要求」完成提交，结束时应尽量保证工作区干净。

## 委托子代理

将任务委托给该子代理，提示词须传达上述「提交要求」。

### 选择代理工具

用户可以选择子代理工具，未选择时按默认顺序检测工具是否可用，使用第一个可用工具。

```bash
# Kimi
kimi --prompt "<提示词>"

# Cursor Agent
agent -p "<提示词>" --yolo

# Claude Code
claude -p "<提示词>" --dangerously-skip-permissions
```

用户提供的子代理名称如果不是以上 3 种，就在用户终端配置中找到对应的别名（不存在笔误），结合别名配置和示例命令选择对应的代理。
