---

同步 git 远程并解决冲突，任务完成的标准是本地和远程保持一致（推送到远程）。默认由主代理直接完成；用户指定子代理时，委托子代理完成。

## 主代理直接同步（默认）

未指定子代理时，由主代理按以下原则执行：

- 优先通过 rebase 保持线性历史
- 禁止直接丢弃提交内容，必须根据提交时间理解修改意图

## 委托子代理同步

用户指定子代理时，将同步任务委托给该子代理（仅在用户明确指定子代理时启用委托）。

### 选择代理工具

用户可以选择子代理工具，未选择时按默认顺序检测工具是否可用，使用第一个可用工具。

```bash
# Kimi
kimi --prompt "<提示词>"

# Cursor Agent
agent -p "<提示词>" --yolo

# Claude Code
claude -p "<提示词>" --dangerously-skip-permissions
```

用户提供的子代理名称如果不是以上 3 种，就在用户终端配置中找到对应的别名，结合别名配置和示例命令选择对应的代理。

### 提示词要求

提示词自由发挥，但任务需满足以下要求：

- 优先通过 rebase 保持线性历史，确保本地和远程保持一致（推送到远程）
- 禁止直接丢弃提交内容，必须根据提交时间理解修改意图
- 冲突需先理解双方修改意图再解决，必要时征求用户意见
